# TCREI Prompt Frame: Meta-Capability – Model Thinking

## Purpose
This prompt frame is designed to strengthen the meta-capability of model thinking: the ability to recognize, compare, construct, test, and integrate models used to understand systems, decisions, causality, and reality.

The frame uses the TCREI structure:
- T = Task
- C = Context
- R = Reference
- E = Evaluate
- I = Iterate

---

# 1. Core Meta-Capability: Model Thinking

Model thinking is the ability to:
- Detect the implicit model behind an idea or decision
- Distinguish between descriptive, predictive, causal, and normative models
- Compare competing models
- Understand model boundaries, assumptions, variables, and failure modes
- Select the correct model for the correct scale and context
- Build multi-model understanding rather than single-model dependence

Typical model categories:
- Mental models
- System models
- Causal models
- Statistical models
- Economic models
- Biological models
- Strategic models
- Organizational models
- Decision models
- Feedback-loop models

# 3. Master TCREI Prompt Frame

```text
TASK
Analyze the following subject using model thinking.

Subject: [Insert subject]

1. Identify the explicit and implicit models being used.
2. Classify each model:
   - descriptive
   - predictive
   - causal
   - normative
3. Define:
   - system boundaries
   - assumptions
   - inputs
   - mechanisms
   - outputs
   - constraints
4. Explain what the model includes and excludes.
5. Identify where the model may fail.
6. Compare at least two alternative models.
7. Determine which model is most appropriate at different scales:
   - individual
   - organizational
   - societal
   - global
8. Build a higher-order integrated model.

CONTEXT
The goal is not only to understand the topic, but to understand the structure of thinking used to interpret the topic.

Focus on:
- causal logic
- feedback loops
- nonlinear effects
- emergence
- time horizons
- uncertainty
- hidden variables
- tradeoffs

REFERENCE
Use the following model categories when relevant:
- first principles
- systems thinking
- feedback loops
- second-order thinking
- game theory
- incentive structures
- network effects
- statistical thinking
- complexity theory
- evolutionary dynamics
- decision theory
- cybernetics

EVALUATE
Evaluate the quality of the model using these criteria:
- explanatory power
- predictive accuracy
- simplicity
- robustness
- adaptability
- scalability
- falsifiability
- sensitivity to assumptions

For each model, score:
- Strengths
- Weaknesses
- Blind spots
- Failure conditions

ITERATE
Improve the model by:
- removing weak assumptions
- adding missing variables
- including feedback loops
- testing against edge cases
- integrating alternative models
- refining the level of analysis

Output Version 1, Version 2, and Version 3 of the improved model.
```

# 4. Specialized Prompt Frames By Folder

## 4.1 Model Recognition

```text
TASK
Read the following argument, decision, policy, or explanation.

Identify:
- the underlying model
- assumptions that are not stated
- variables that drive the model
- what the model assumes is causal
- what evidence would disprove the model

CONTEXT
Many people use models unconsciously. The goal is to surface the hidden structure of reasoning.

REFERENCE
Possible hidden models include:
- linear cause-effect
- market equilibrium
- optimization
- hierarchy
- feedback loop
- competition
- evolutionary selection
- scarcity
- rational actor

EVALUATE
Determine:
- which assumptions are strongest
- which assumptions are weakest
- what the model ignores

ITERATE
Rewrite the original reasoning using a stronger and more explicit model.
```

---

## 4.2 Model Construction

```text
TASK
Construct a model for the following system:
[Insert system]

Define:
- purpose of the system
- components
- inputs
- processes
- outputs
- constraints
- feedback loops
- time delays
- nonlinear interactions

CONTEXT
The objective is to produce a working explanatory model rather than a list of observations.

REFERENCE
Use this structure:
1. System boundary
2. Internal variables
3. External variables
4. Interactions
5. Reinforcing loops
6. Balancing loops
7. Failure points

EVALUATE
Assess whether the model:
- explains observed behavior
- predicts future behavior
- remains stable under change

ITERATE
Produce:
- simple model
- intermediate model
- advanced multi-layer model
```

---

## 4.3 Model Comparison

```text
TASK
Compare Model A and Model B for understanding:
[Insert topic]

For each model explain:
- assumptions
- causal logic
- strengths
- limitations
- appropriate scale
- failure conditions

CONTEXT
No model is universally correct. Different models perform better under different conditions.

REFERENCE
Compare dimensions such as:
- simple vs complex
- linear vs nonlinear
- static vs dynamic
- deterministic vs probabilistic
- centralized vs distributed

EVALUATE
Determine:
- which model explains more
- which model predicts better
- which model is more robust under uncertainty

ITERATE
Create a synthesized hybrid model that combines the strongest parts of both.
```

# 5. Meta-Level Reflection Layer

Use this layer after every model-thinking exercise.

```text
1. What model was I using without realizing it?
2. Which assumptions shaped my interpretation?
3. What scale was I analyzing at?
4. What important variables were excluded?
5. Which alternative model changes the conclusion most?
6. Where does the model break down?
7. What higher-level model contains this model?
8. What would a systems scientist, economist, biologist, engineer, and philosopher each see differently?
```

# 7. Highest-Level Meta-Capability

The deepest form of model thinking is not merely selecting a model.

It is understanding:
- why a particular model appears convincing
- what reality that model makes visible
- what reality that model hides
- how multiple models interact
- how models themselves evolve over time

At the highest level, model thinking becomes meta-model thinking: the ability to think about the architecture of thinking itself.
