# TCREI Prompt Frame: Meta-Capability – Model Thinking

## Purpose
This prompt frame is designed to strengthen the meta-capability of model thinking: the ability to recognize, compare, construct, test, and integrate models used to understand systems, decisions, causality, and reality.

The frame uses the TCREI structure:
- T = Task
- C = Context
- R = Reference
- E = Evaluate
- I = Iterate

---

# 1. Core Meta-Capability: Model Thinking

Model thinking is the ability to:
- Detect the implicit model behind an idea or decision
- Distinguish between descriptive, predictive, causal, and normative models
- Compare competing models
- Understand model boundaries, assumptions, variables, and failure modes
- Select the correct model for the correct scale and context
- Build multi-model understanding rather than single-model dependence

Typical model categories:
- Mental models
- System models
- Causal models
- Statistical models
- Economic models
- Biological models
- Strategic models
- Organizational models
- Decision models
- Feedback-loop models

# 3. Master TCREI Prompt Frame

```text
TASK
Analyze the following subject using model thinking.

Subject: [Insert subject]

1. Identify the explicit and implicit models being used.
2. Classify each model:
   - descriptive
   - predictive
   - causal
   - normative
3. Define:
   - system boundaries
   - assumptions
   - inputs
   - mechanisms
   - outputs
   - constraints
4. Explain what the model includes and excludes.
5. Identify where the model may fail.
6. Compare at least two alternative models.
7. Determine which model is most appropriate at different scales:
   - individual
   - organizational
   - societal
   - global
8. Build a higher-order integrated model.

CONTEXT
The goal is not only to understand the topic, but to understand the structure of thinking used to interpret the topic.

Focus on:
- causal logic
- feedback loops
- nonlinear effects
- emergence
- time horizons
- uncertainty
- hidden variables
- tradeoffs

REFERENCE
Use the following model categories when relevant:
- first principles
- systems thinking
- feedback loops
- second-order thinking
- game theory
- incentive structures
- network effects
- statistical thinking
- complexity theory
- evolutionary dynamics
- decision theory
- cybernetics

EVALUATE
Evaluate the quality of the model using these criteria:
- explanatory power
- predictive accuracy
- simplicity
- robustness
- adaptability
- scalability
- falsifiability
- sensitivity to assumptions

For each model, score:
- Strengths
- Weaknesses
- Blind spots
- Failure conditions

ITERATE
Improve the model by:
- removing weak assumptions
- adding missing variables
- including feedback loops
- testing against edge cases
- integrating alternative models
- refining the level of analysis

Output Version 1, Version 2, and Version 3 of the improved model.
```

# 4. Specialized Prompt Frames By Folder

## 4.1 Model Recognition

```text
TASK
Read the following argument, decision, policy, or explanation.

Identify:
- the underlying model
- assumptions that are not stated
- variables that drive the model
- what the model assumes is causal
- what evidence would disprove the model

CONTEXT
Many people use models unconsciously. The goal is to surface the hidden structure of reasoning.

REFERENCE
Possible hidden models include:
- linear cause-effect
- market equilibrium
- optimization
- hierarchy
- feedback loop
- competition
- evolutionary selection
- scarcity
- rational actor

EVALUATE
Determine:
- which assumptions are strongest
- which assumptions are weakest
- what the model ignores

ITERATE
Rewrite the original reasoning using a stronger and more explicit model.
```

## 4.2 Model Construction

```text
TASK
Construct a model for the following system:
[Insert system]

Define:
- purpose of the system
- components
- inputs
- processes
- outputs
- constraints
- feedback loops
- time delays
- nonlinear interactions

CONTEXT
The objective is to produce a working explanatory model rather than a list of observations.

REFERENCE
Use this structure:
1. System boundary
2. Internal variables
3. External variables
4. Interactions
5. Reinforcing loops
6. Balancing loops
7. Failure points

EVALUATE
Assess whether the model:
- explains observed behavior
- predicts future behavior
- remains stable under change

ITERATE
Produce:
- simple model
- intermediate model
- advanced multi-layer model
```

---

## 4.3 Model Comparison

```text
TASK
Compare Model A and Model B for understanding:
[Insert topic]

For each model explain:
- assumptions
- causal logic
- strengths
- limitations
- appropriate scale
- failure conditions

CONTEXT
No model is universally correct. Different models perform better under different conditions.

REFERENCE
Compare dimensions such as:
- simple vs complex
- linear vs nonlinear
- static vs dynamic
- deterministic vs probabilistic
- centralized vs distributed

EVALUATE
Determine:
- which model explains more
- which model predicts better
- which model is more robust under uncertainty

ITERATE
Create a synthesized hybrid model that combines the strongest parts of both.
```

# 5. Meta-Level Reflection Layer

Use this layer after every model-thinking exercise.

```text
1. What model was I using without realizing it?
2. Which assumptions shaped my interpretation?
3. What scale was I analyzing at?
4. What important variables were excluded?
5. Which alternative model changes the conclusion most?
6. Where does the model break down?
7. What higher-level model contains this model?
8. What would a systems scientist, economist, biologist, engineer, and philosopher each see differently?
```

# 7. Highest-Level Meta-Capability

The deepest form of model thinking is not merely selecting a model.

It is understanding:
- why a particular model appears convincing
- what reality that model makes visible
- what reality that model hides
- how multiple models interact
- how models themselves evolve over time

At the highest level, model thinking becomes meta-model thinking: the ability to think about the architecture of thinking itself.

# 8. Applied Example: Subject = CEO

```text
TASK
Analyze the role of a CEO using model thinking.

Subject: CEO

1. Identify the models commonly used to understand a CEO:
   - heroic leader model
   - rational decision-maker model
   - systems orchestrator model
   - capital allocator model
   - political coalition manager model
   - cultural architect model
   - adaptive complex-systems navigator model

2. Classify each model:

- Heroic leader = normative + descriptive
- Rational decision-maker = predictive + causal
- Systems orchestrator = causal + systems model
- Capital allocator = economic + decision model
- Political coalition manager = game-theoretic + network model
- Cultural architect = behavioral + social model
- Complex-systems navigator = adaptive + nonlinear model

3. Define system boundaries:

Internal boundary:
- board
- executive team
- employees
- capital
- operations
- information flows

External boundary:
- customers
- regulators
- investors
- competitors
- technology shifts
- macroeconomy
- society

4. Inputs
- market signals
- financial data
- organizational capabilities
- stakeholder incentives
- risk indicators
- technological change
- regulatory pressure

5. Transformation Mechanisms
- interpretation of information
- strategic prioritization
- resource allocation
- decision-making
- communication
- coordination
- culture shaping

6. Outputs
- strategy
- organizational behavior
- capital deployment
- innovation rate
- financial performance
- resilience
- reputation

7. Constraints
- bounded rationality
- limited information
- time pressure
- political conflict
- capital constraints
- legal obligations
- cognitive bias

CONTEXT
The CEO should not be analyzed only as an individual person. The CEO is a functional node within a multi-layer system connecting:
- individual cognition
- organizational structure
- market dynamics
- societal expectations

REFERENCE
Use these lenses:
- systems thinking
- agency theory
- game theory
- network effects
- cybernetics
- complexity theory
- second-order consequences

EVALUATE
Compare the explanatory power of the following CEO models:

1. Heroic Leader Model
Strengths:
- explains charisma and symbolic influence
Weaknesses:
- over-attributes outcomes to one person
Blind spots:
- ignores systems and structural forces
Failure condition:
- breaks down in complex distributed organizations

2. Capital Allocator Model
Strengths:
- strong for finance, portfolio, and investment decisions
Weaknesses:
- underestimates culture and human factors
Failure condition:
- weak in innovation-driven or knowledge-intensive firms

3. Systems Orchestrator Model
Strengths:
- captures interactions across functions and stakeholders
Weaknesses:
- more difficult to operationalize and measure
Failure condition:
- weak when rapid unilateral decisions are required

4. Complex Adaptive Navigator Model
Strengths:
- strongest under uncertainty, disruption, and nonlinear change
Weaknesses:
- can become too abstract
Failure condition:
- weak in stable and highly regulated environments

ITERATE
Version 1:
CEO as decision-maker

Version 2:
CEO as allocator of capital, incentives, and attention

Version 3:
CEO as an adaptive systems integrator operating across multiple scales:
- person
- team
- organization
- market
- civilization

META-REFLECTION
1. Which CEO model is being assumed?
2. What does that model explain well?
3. What does it ignore?
4. Under what conditions does another model become superior?
5. How would the analysis change if the CEO were viewed primarily as:
   - a systems architect
   - an information-processing node
   - a political actor
   - a cultural signal generator
```

# 9. Applied Example: Subject = CFA

```text
TASK
Analyze the CFA using model thinking.

Subject: CFA

1. Identify the models commonly used to understand a CFA:
   - financial analyst model
   - probabilistic decision-maker model
   - valuation model builder
   - risk manager model
   - market interpreter model
   - portfolio systems model
   - behavioral-finance model

2. Classify each model:

- Financial analyst = descriptive + predictive
- Probabilistic decision-maker = statistical + causal
- Valuation model builder = economic + quantitative
- Risk manager = systems + uncertainty model
- Market interpreter = adaptive + informational model
- Portfolio systems model = optimization + dynamic model
- Behavioral-finance model = psychological + nonlinear model

3. Define system boundaries:

Internal boundary:
- investment process
- research assumptions
- valuation framework
- risk limits
- portfolio objectives
- time horizon

External boundary:
- capital markets
- macroeconomy
- regulation
- interest rates
- technology
- investor behavior
- geopolitical events

4. Inputs
- financial statements
- market prices
- earnings expectations
- macroeconomic data
- industry structure
- management quality
- risk indicators

5. Transformation Mechanisms
- financial analysis
- scenario modeling
- probability assessment
- valuation
- portfolio construction
- risk adjustment
- interpretation of signals

6. Outputs
- investment thesis
- intrinsic value estimate
- expected return
- portfolio allocation
- risk-adjusted decision
- buy, hold, or sell conclusion

7. Constraints
- incomplete information
- market noise
- uncertainty
- cognitive bias
- time pressure
- liquidity constraints
- model error

CONTEXT
The CFA should not be understood only as a technical credential. The CFA functions as a model-based interpreter between information, markets, risk, and decision-making.

REFERENCE
Use these lenses:
- discounted cash flow
- portfolio theory
- behavioral finance
- Bayesian reasoning
- systems thinking
- game theory
- second-order effects
- complexity and nonlinear market dynamics

EVALUATE
Compare the explanatory power of the following CFA models:

1. Valuation Model Builder
Strengths:
- strong for estimating intrinsic value and long-term return
Weaknesses:
- highly sensitive to assumptions
Blind spots:
- underestimates market psychology and regime change
Failure condition:
- breaks down during structural discontinuities or crises

2. Risk Manager Model
Strengths:
- strong for downside protection and portfolio resilience
Weaknesses:
- may reduce upside by excessive conservatism
Blind spots:
- may miss nonlinear positive opportunities
Failure condition:
- weak when historical risk relationships no longer hold

3. Behavioral-Finance Model
Strengths:
- explains irrational pricing, herding, and sentiment
Weaknesses:
- difficult to quantify precisely
Blind spots:
- may underweight fundamentals
Failure condition:
- weak in highly efficient and stable markets

4. Portfolio Systems Model
Strengths:
- captures interaction among assets, incentives, and constraints
Weaknesses:
- requires many variables and assumptions
Failure condition:
- weak when correlation structures change suddenly

ITERATE
Version 1:
CFA as financial analyst producing forecasts and valuations

Version 2:
CFA as decision architect balancing return, risk, and uncertainty

Version 3:
CFA as a multi-model integrator operating across:
- company
- industry
- market
- macroeconomic system
- global capital flows

META-REFLECTION
1. Which investment model is being assumed?
2. What variables does the model emphasize?
3. Which variables are ignored?
4. When does a valuation model become less useful than a behavioral or systems model?
5. How would the analysis change if the CFA were viewed primarily as:
   - a probabilistic thinker
   - a systems analyst
   - a risk allocator
   - a behavioral interpreter
```

Additional analytical structures relevant to CFA model thinking:

```text
Cross-Scale Analysis
- Micro scale: company fundamentals
- Meso scale: industry structure and competition
- Macro scale: interest rates, inflation, GDP, policy
- Global scale: capital flows, geopolitics, technological change

Feedback Loops
- Positive loop: rising prices -> stronger sentiment -> more inflows -> higher prices
- Negative loop: falling earnings -> lower confidence -> reduced investment -> lower earnings

Nonlinear Dynamics
- Small changes in discount rates can produce large changes in valuation.
- Market behavior may shift abruptly at critical thresholds such as liquidity stress or policy changes.

Phase Transitions
- Bull market -> correction -> crisis -> recovery
- Stable equilibrium -> unstable volatility regime

Emergence
- Individual investor decisions aggregate into market trends.
- Market trends aggregate into macroeconomic cycles.
```
